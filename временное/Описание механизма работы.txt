Блокчейн используется как хранилище для контроля версий файлов.
Данные о текущем рабочем состоянии хранятся в файле состояния.
В рабочее состояние входит:
	Идентификатор коммита.
	Контрольные суммы текущих файлов.
	(?)Копия всех текущих контролируемых файлов.
	Контрольные суммы добавленных файлов.
	Имена удаляемых файлов.
	Имена файлов, которые должны быть зафиксированны в следующем коммите.
Данные о доверенных лицах (имеющих право создавать коммиты) хранятся в файле конфигурации.
	Эти данные используются при совместном использовании через репозиторий.
	Эти данные хранятся в репозитории.
Данные о игнорируемых файлах берутся из файла ".ignore".


Планируемые комманды:
init - создание нового репозитория (включение контроля версий) в текущей дериктории.
add - добавления файла в контроль версий.
rm - удаление файла из контроля версий.
commit - фиксация изменений.
setremote - установка удалённого репозитория и синхронизация с ним.
unsetremote - отключение удалённого репозитория.
sync - вызывает синхронизацию с удалённым репозиторием.



Описание работы:
При создании нового репозитория (команда init) создаётся папка системы контроля версий.
В данной папке создаются файлы:
	Блокчейна.
	Рабочего состояния.
Первичное рабочее состояние имеет нулевой идентификатор коммита и не содержит контрольных сумм.
